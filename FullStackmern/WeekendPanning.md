# Weekend planning

## Context :

You and your friends are always excited about the weekends, but planning them is often a nightmare. Coordinating everyone's schedules, deciding on fun activities, and finding reliable information about events and venues usually ends up in endless group chats, confusion, and ultimately, cancelled plans. Frustrated by the constant chaos, you decide to take matters into your own hands. With your tech-savvy skills and a passion for making life easier, you set out to develop an innovative application that will revolutionise the way weekends are planned. Your goal is to create a platform that not only simplifies the process but also enhances the overall experience of spending quality time with friends.

## Project Goal

- The UI/UX should be best in class
- Develop an application that addresses the common challenges faced when planning weekends with friends.
- The application should aim to make the process easier by providing features that help with scheduling, finding activities, and managing logistics.
- It should also facilitate better communication and decision-making among group members.

## Minimum Expected Features:

- **User Authentication :**  use services like Firebase or supabase for user authentication.
- **Group Scheduling**:
    - Implement a feature that allows users to share their availability and find common free times for the group.
    - Use the scheduling greedy algorithm with custom hooks
    - Integrate with calendar applications to sync and manage schedules.
    - Use - Google calendar API to sync it with google calendar.
- **Activity Suggestions**:
    - Provide a curated list of events, activities, and venues based on user preferences and location using graph data structure is a must.
    - Include details such as opening hours, prices, and user reviews.
- **Polls and Voting**:
    - Allow users to create polls for different activities or venues and vote on their preferences.
    - Display results to help the group make decisions easily.
- **Notification Management**:
    - Provide reminders and notifications to keep everyone informed and on track.
- **Admin Side**:
    - **User Management**: Handle user profiles, group settings, and permissions.
    - **Event Analytics Dashboard**: Provide insights into popular activities, user engagement, and feedback.
    - **Support and Feedback Management**: Offer a platform for user support and gathering feedback to improve the application.
- **Database solution :** You need to use database solution like Firebase , Supabase for this whole application.
- **Market-Beating Feature**: Implement a social component where users can connect with other users planning similar activities, allowing for larger gatherings and new social connections. using the graph data structure to maintain the connections between them.

## Submission Guideline:

- A functional web application with source code.
- Comprehensive documentation including architecture, API details, and user guide.
- A presentation demonstrating the application and highlighting how it improves the weekend planning experience over existing solutions.

## Project and Team Management Guide:

- First, decide amongst yourselves who will be the team lead - this will be the most responsible person in your team.
    - This person would be responsible for the following tasks:
        - Creating and assigning tasks to group members, managing Github projects.
        - People management, overall completion of the project within the deadline.
        - There should be a daily scrum where the team lead ensures that the assigned tasks are implemented on time.
- During the first hours, you need to focus on market research and come up with an outline for all your design guidelines (color schemes, typography, visual elements, animations, and visual assets like images, gifs, videos, etc.).
- Break down all these as tasks and create a structured plan on how you are going to execute this project within the given deadline.
- Now respective team members can start working on their assigned tasks and features.
